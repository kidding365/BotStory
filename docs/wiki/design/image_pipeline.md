# Image Generation Pipeline

The system uses a structured prompt construction method to ensure visual consistency across turns.

## Prompt Construction
The Storyteller AI does not just send a "prompt"; it fills a template:

- **Characters:** `[imageStyleCharacterPre]` + `[illustrAppearance]` + `[illustrClothes]` + `[imageStyleCharacterPost]`
- **Settings:** `[imageStyleNonCharacterPre]` + `[illustrSetting]` + `[imageStyleNonCharacterPost]`

## Dynamic Elements
The Storyteller AI updates these fields every turn:
- `illustrSubject`: The main focus of the image.
- `illustrAppearance`: Updated if the character changes (e.g., gets wounded).
- `illustrClothes`: Updated if the character changes clothes.
- `illustrSetting`: Updated based on the current location.

## BotStory Implementation
We should implement a similar template system where the "Style" is a global world setting, but the "Details" are dynamically generated by the AI.
